# Підтримка Arduino Micro для енкодера

## Зміни

Додано підтримку Arduino Micro в модулі керування енкодером KY-040.

### Проблема
Функція `readAB_fast()` використовувала прямі біти регістру `PIND` для Arduino UNO:
- D2 = PD2 (bit 2)
- D3 = PD3 (bit 3)

На Arduino Micro ці піни мають інші біти:
- D2 = PD1 (bit 1)
- D3 = PD0 (bit 0)

### Рішення
Додано умовну компіляцію `#if defined(ARDUINO_AVR_MICRO)` для автоматичного визначення плати та використання правильних бітів.

### Змінені файли
- `encoder_k040.cpp` - додано підтримку Micro в `readAB_fast()`
- `encoder_k040.h` - оновлено коментарі
- `mql_2004_I2C_encoder_V2.ino` - додано коментарі про сумісність HARD керування

### Перевірка інших функцій
- ✅ **HARD START override** (рядки 251-261) - використовує `digitalRead()`, працює на обох платах
- ✅ **HARD DIA accel override** (рядки 263-277) - використовує `digitalRead()`, працює на обох платах
- ✅ Інші піни (A0-A3, D10-D12) однакові для обох плат

### Сумісність
- ✅ Arduino UNO (без змін)
- ✅ Arduino Nano (без змін)
- ✅ Arduino Micro (нова підтримка)

### Технічні деталі
- Використовується `ARDUINO_AVR_MICRO` для визначення плати
- Інші піни (A0-A3, D10-D12) однакові для обох плат
- `digitalPinToInterrupt()` працює коректно на обох платах
